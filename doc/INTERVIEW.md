# 面试速记卡 — 强化学习(基于 Pac-Man 项目)

> 一句话项目定位:用**近似 Q-learning**训练 AI 自动玩 Pac-Man,通关率从 ~43% 提到 ~58%。

---

## 0. 面试官问"你有没有强化学习经历?" —— 标准回答

**30秒版(先说这个):**

> "有。我做过一个 Pac-Man 项目,用**强化学习**训练 AI 自动玩游戏。用的是**近似 Q-learning**(线性函数逼近),因为状态空间太大没法用查表法。我自己实现了训练环境、TD 学习更新和奖励设计,最后通过**诊断失败原因 → 针对性改进奖励和特征**,把通关率从 43% 提到约 58%。"

**展开版(如果让你详细说,按 STAR):**

1. **背景/难点**:Pac-Man 状态空间是天文数字(位置×所有鬼位置×每颗豆),查表法存不下 → 改用线性近似 `Q(s,a)=w·f(s,a)`,只学几个权重。
2. **方法**:时序差分(TD)学习,ε-greedy 探索,在 3 张真实地图上训练 3 万局。
3. **亮点(重点讲这个)**:基线卡在 43%,我写诊断工具发现**100% 失败都是被鬼抓死**;于是想到吃大豆能让鬼变可吃、消除死因 → 加大豆奖励 + 大豆特征 + 前瞻搜索 → 58%。
4. **诚实点**:也试过 DQN 神经网络,反而更差(20%),体会到**"复杂模型 ≠ 更好",奖励设计和问题分析更关键**。

**收尾金句:**
> "这个项目让我理解:强化学习真正难的不是套公式,而是**奖励设计**和**用数据诊断 agent 为什么失败**。"

---

## 1. 核心公式(必背)

**近似 Q 值(为什么不用查表):**
```
Q(s,a) = Σ_i w_i · f_i(s,a) = wᵀ·f(s,a)
```
只学 n 个权重 w_i,而非天文数字个表项。f_i 是手工特征。

**TD 学习三步(核心):**
```
① TD目标:   y = r + γ · max_a' Q(s',a')      (终止则 y = r)
② TD误差:   δ = y − Q(s,a)
③ 更新权重:  w_i ← w_i + α · δ · f_i(s,a)
```

**三步的意义:**
- ① y:这一步的"更准答案" = 即时奖励 r + 折扣后的未来最优价值
- ② δ:预测误差 = "惊讶程度"(>0惊喜该调高,<0失望该调低)
- ③ 把误差按特征贡献 f_i 分摊回每个权重(本质是梯度下降)

**参数:** γ=0.95(折扣),α=0.0008(学习率),ε:1.0→0.05(探索衰减)

---

## 2. r(奖励)怎么来 —— 环境按规则打分,不是学的

| 事件 | r |
|---|--:|
| 每走一步 | −1(催快点) |
| 吃普通豆 | +10 |
| 吃大豆 | +200 |
| 吃受惊鬼 | +200 |
| 通关 | +500 |
| 被抓死 | −500 |

**关键:** r 是即时的(只反映当下),未来靠 ① 里的 γ·maxQ 补;这些数值是**我们设计的(reward shaping)**,目的是引导 AI 学出想要的行为。

---

## 3. RL 本质 = 试错学习(对比监督学习)

- 监督学习:有标准答案标签。
- 强化学习:**没标签,只有延迟奖励**,agent 自己玩几万局、试错摸索哪些动作长期最优。
- ⚠️ 不是"放它瞎走就成才":**七分靠奖励/特征设计,三分靠它自己走**。证据:训练次数不变,光改"鼓励吃大豆"就从 43%→58%。
- 需要 ε-greedy 探索,否则困在局部最优。

---

## 4. on-policy vs off-policy(高频考点)

**两个策略:**
- 行为策略(实际怎么走/收集数据)= ε-greedy
- 目标策略(想学成什么)= 贪心

| | 定义 | 例子 |
|---|---|---|
| **off-policy** | 行为 ≠ 目标(学一套跑一套) | **Q-learning**(更新用 max) |
| **on-policy** | 行为 = 目标 | SARSA(更新用实际选的 a') |

**Q-learning 为什么 off-policy:** 更新用 `max_a' Q(s',a')`(假设走最优),但实际用 ε-greedy 会随机探索,两者不一致。

**为什么这个项目选 off-policy:**
1. 目标是学最优,部署时用纯贪心不探索 → 正好匹配"学一套跑一套"
2. off-policy 能复用数据、样本效率高(DQN 经验回放的前提)
3. Pac-Man 是模拟器,探索失误没代价,不需要 SARSA 的保守

**什么时候用 on-policy:** 探索代价高(机器人/自动驾驶)、追求训练稳定性(PPO 就是 on-policy,RLHF 训 ChatGPT 用它)。

**SARSA 公式对比:** `y = r + γ·Q(s',a')`,a' 是实际(ε-greedy)选的,不是 max。

## 5. 9 个特征(展示策略可解释)

| 特征 | 权重 | 含义 |
|---|--:|---|
| 旁边1格有危险鬼 | −529 | 最怕贴脸鬼 |
| 吃到豆 | +63 | 鼓励吃 |
| 到最近豆BFS距离 | −29 | 朝豆走 |
| 到受惊鬼距离 | +29 | 去吃得分 |
| 2格内危险鬼数 | −99 | 提前预警 |
| 吃到大豆 | +154 | 主动吃大豆 |
| 被追时靠近大豆 | +228 | 用大豆化险 |
| 离鬼安全距离(有界) | +106 | 尽量远离鬼 |

> 权重正负可解释 = 线性模型相对神经网络的最大优势。

---

## 6. 三个增强(43%→58% 的关键)

1. **鼓励吃大豆**(诊断驱动:100%死于碰撞 → 吃大豆让鬼变可吃)
2. **深度3前瞻搜索**(玩时展开3步,叶子用学到的Q评估)
3. **防循环**(记最近12格,回头扣分,打破来回踱步)

## 7. 常见追问速查

| 追问 | 答 |
|---|---|
| Q-learning vs 监督学习 | 监督有标签;RL 只有延迟奖励,试错学习 |
| γ 是什么 | 未来奖励折扣,0.95 兼顾眼前与长远 |
| ε-greedy 为什么 | 平衡探索与利用,避免局部最优 |
| off-policy 好处 | 复用数据、样本效率高、探索与学习解耦 |
| off-policy 坏处 | 训练不稳,配神经网络易发散,需 target network |
| DQN 是 on 还是 off | off-policy,所以能用经验回放 |
| DQN vs 线性 | 神经网络表达力强但难调不可解释;线性可解释 |
| 你的方法局限 | 依赖深度3前瞻;线性表达力有限是天花板 |
